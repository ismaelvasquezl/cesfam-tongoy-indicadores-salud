Diagnóstico y depuración para GitHub Pages

Esta sección está pensada para prevenir los errores más frecuentes al publicar un sitio estático gratuito, especialmente los relacionados con Jekyll, rutas rotas y configuración de rama o carpeta de despliegue.

Error 1: Jekyll intenta compilar el sitio

  • Solución: agrega .nojekyll en la raíz del repositorio.
  • Útil cuando el proyecto es HTML/CSS/JS puro y no usa SCSS ni plantillas.
  • Evita fallos como búsquedas de /docs o compilaciones Sass innecesarias.

Error 2: 404 por carpetas o rutas

  • Usa enlaces relativos simples: ./mmss-abril.html.
  • Configura GitHub Pages en main y / (root) si tus HTML están en raíz.
  • Agrega un 404.html propio para orientar al usuario si una ruta falla.

Error 3: archivos estáticos no cargan

  • Evita rutas absolutas que comiencen con / si el repositorio no es usuario.github.io.
  • Prefiere ./styles/archivo.css o CSS embebido mientras estabilizas el sitio.
  • Verifica mayúsculas y minúsculas exactas en nombres de archivo.
